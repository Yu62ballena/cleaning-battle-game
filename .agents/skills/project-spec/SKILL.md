# お片付け大戦争 プロジェクト仕様

---

## 世界観・ビジュアルの方向性（最重要・妥協禁止）

このゲームは「プロトタイプだから見た目は簡素でいい」という発想を明確に否定する。**実装する範囲(スコープ)は絞ってよいが、実装する部分の見た目のクオリティは最初から本気で作り込むこと。** 単色の矩形をキャラクターやおもちゃとして扱うことは禁止する。

### 参考にしたい作品と、それぞれから汲み取ってほしい要素

- **Tools Up! / Moving Out**：デフォルメされた、丸みのあるトゥーン調のキャラクターデザイン。手足が短くコロンとした体型で、表情豊か。家の中の生活感のある空間(家具・小物にリアルな質感がありつつも親しみやすい丸み)
- **塊魂（Katamari Damacy）**：家の中にあるあらゆる物(おもちゃ・文房具・生活用品)が、彩度高めのビビッドな色使いで、どこか陽気で愛嬌のある佇まいをしている。物同士のルール無用な混沌とした散らかり方自体が「楽しそう」に見える空気感。おもちゃの一つ一つに、単なる四角ではなく「らしさ」(車ならタイヤ、ぬいぐるみなら耳、のような特徴的なパーツ)を持たせる

### 実装方針

外部画像アセットが用意できていない場合でも、**PhaserのGraphics APIを使って、以下の要素を必ず含めた「描画コード」でキャラクター・おもちゃ・家具を表現すること**：

- 角の丸み（矩形ではなく、角丸・楕円・複数図形の組み合わせで輪郭を作る）
- 簡単な表情（目・口など、最低限のパーツで「顔」を感じさせる）
- 立体感を出す簡易的な陰影（グラデーションまたは影用の暗い色レイヤーを底面に配置）
- キャラクターやおもちゃごとの「らしさ」を出すシルエットの工夫（例：親キャラは掃除機のノズルを模した形、大きいおもちゃはぬいぐるみの耳や車のタイヤなど、対象物の特徴を捉えたパーツ）
- ビビッドで彩度高めの配色（下記カラーパレットをベースに、単調にならないよう複数トーンを組み合わせる）

「まず動けばいい、見た目は後で」という判断は行わない。動く・遊べることと、見た目が作り込まれていることは、同時に満たすべき条件として扱うこと。

## コンセプト

親子（1人vs1人）が1台のキーボードを分割して対戦するローカルゲーム。9歳の子どもでも直感的に操作・理解できることを最優先する。「誰が今どちらのキャラで、何点入っているか」が一瞬で分かるUIにする。舞台は家の中。散らかった部屋そのものが、塊魂的などこか楽しげな空気をまとっていること。

## 画面構成（Step1範囲）

1. **タイトル画面**：「スタート」ボタンのみ。操作方法（矢印キー＝親／WASD＝子ども）を画面に表示。背景にも上記ビジュアル方針を適用し、単なる無地の背景にしない
2. **ゲーム画面**：クォータービュー風の見下ろし2D画面。上部に「残り時間」「親の得点」「子どもの得点」を常時表示
3. **結果画面**：勝敗表示＋「もう一度あそぶ」ボタン

## タイポグラフィ

- 見出し・スコア数字：太めのゴシック体（`font-weight: 700`以上）。Next.js標準のシステムフォントで可（`font-sans`）
- スコア数字は大きめ（24px以上）にし、離れた位置からでも読める大きさにする
- 操作説明文は16px以上

## UIコンポーネント方針

- ボタン（スタート・もう一度あそぶ）：角丸12px以上、影を軽くつけて押せる感を出す。ホバー/タップで少し縮小するアニメーション
- スコアバー：画面上部に固定。左＝親（青背景）、右＝子ども（オレンジ背景）、中央＝残り時間
- 基地HPゲージ：基地キャラの真上に小さく表示。HPが減ると視覚的にわかるようゲージが縮む

## アニメーション方針

- おもちゃを吸った瞬間：小さく縮小してから消えるアニメーション（0.2秒程度）
- 基地HPが0になった瞬間：基地全体が一瞬光る（フラッシュ）演出
- キャラクターの移動時：わずかな上下バウンス（歩いている・動いている感じを出す簡易アニメーション）を加える

## 機能仕様

### 操作
- 親：矢印キー（↑↓←→）で上下左右移動（8方向可）
- 子ども：WASDキーで上下左右移動（8方向可）

### 親（掃除機）の挙動
- おもちゃに接触すると自動で吸引する（ボタン操作不要。接触判定のみでよい）
- 容量上限（10）を超える場合は吸えない
- 吸ったおもちゃはインベントリとして保持し、移動速度は下記の計算式に従って低下する
- ゴミ箱に接触すると、保持しているおもちゃがすべて得点化され、インベントリが空になり容量・重さがリセットされる（移動速度も基本速度に戻る）

### 子ども（おもちゃ側）の挙動
- 床にあるおもちゃは、時間経過で自動的に得点としてカウントされ続ける（「床にある間だけ得点になる」という原則を必ず守ること）
- Step1では移動のみ（散らかす操作・妨害操作は実装しない。すでに散らかっている状態からスタートする）

### 基地（おもちゃ箱）
- マップ上に固定配置。中に初期状態でおもちゃ数個（大小混在）を格納した状態でスタートする
- 基地内のおもちゃは、基地が「開放」状態になるまで親は吸えない
- 親が基地に隣接した状態を継続すると、基地HPが1秒あたり10減少する
- 基地HPが0になったら「開放」状態になり、以後は通常のおもちゃと同様に親が吸える
- 修理キットを持った子どもが基地に隣接すると、基地HPが1秒あたり20回復する（上限100）
- 基地HPは画面上、基地の真上にゲージとして表示する

### 修理キット
- マップ上に1個、固定位置に配置（Step1範囲。将来的に基地内・フィールドの2箇所案あり、Step2以降で対応）
- 子どもが接触すると「所持」状態になる（見た目にキャラの色を変える等で所持が分かるようにする）
- 所持中に基地に隣接すると上記の回復効果が発生する

### 数値仕様（Step1固定値）

- 制限時間：120秒
- 掃除機の容量上限：10
- おもちゃの種類（Step1は2種類のみ）
  - 小：容量消費1、得点1、重さ寄与1
  - 大：容量消費5、得点10、重さ寄与10
- 移動速度の計算式：`速度 = max(0.2, 1 / (1 + 重さ合計 ÷ 10)) × 基本速度`
  - 基本速度は仮に「1秒間に画面幅の約1/4を移動できる速度」を基準値とする
- 基地HP上限：100（削れ方・回復速度は上記「基地」項目の通り）

### 得点・勝敗
- 時間終了時点で「親の得点（ゴミ箱に運んだ合計）」と「子どもの得点（床にあるおもちゃの合計、その時点のスナップショット）」を比較し、多い方が勝ち
- 結果画面に「親の勝ち／子どもの勝ち／引き分け」と、それぞれの最終得点を表示する

### エッジケース
- 親の容量がちょうど埋まっている状態で、容量ぴったりのおもちゃに接触した場合：吸える（容量オーバーにならない場合はOK）
- 修理キットを持った子どもが基地HP満タンの状態で基地に隣接し続けても、100を超えて回復しない（上限でクランプする）
- 制限時間が0になった瞬間に親または子どもがおもちゃに接触していても、その時点の得点計算は行わない（時間切れの瞬間で計算を打ち切る）
- 親のインベントリが空の状態でゴミ箱に接触しても、何も起こらない（エラーにならないこと）
