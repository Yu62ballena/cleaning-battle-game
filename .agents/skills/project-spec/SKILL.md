# お片付け大戦争 プロジェクト仕様

---

## コンセプト

親子（1人vs1人）が1台のキーボードを分割して対戦するローカルゲーム。9歳の子どもでも直感的に操作・理解できることを最優先する。見た目は簡素でよいが、「誰が今どちらのキャラで、何点入っているか」が一瞬で分かるUIにする。

## 画面構成（Step1範囲）

1. **タイトル画面**：「スタート」ボタンのみ。操作方法（矢印キー＝親／WASD＝子ども）を画面に表示
2. **ゲーム画面**：クォータービュー風の見下ろし2D画面。上部に「残り時間」「親の得点」「子どもの得点」を常時表示
3. **結果画面**：勝敗表示＋「もう一度あそぶ」ボタン

## タイポグラフィ

- 見出し・スコア数字：太めのゴシック体（`font-weight: 700`以上）。Next.js標準のシステムフォントで可（`font-sans`）
- スコア数字は大きめ（24px以上）にし、離れた位置からでも読める大きさにする
- 操作説明文は16px以上

## UIコンポーネント方針

- ボタン（スタート・もう一度あそぶ）：角丸12px、影を軽くつけて押せる感を出す。ホバー/タップで少し縮小するアニメーション
- スコアバー：画面上部に固定。左＝親（青背景）、右＝子ども（オレンジ背景）、中央＝残り時間
- 基地HPゲージ：基地キャラの真上に小さく表示。HPが減ると視覚的にわかるようゲージが縮む

## アニメーション方針

- おもちゃを吸った瞬間：小さく縮小してから消えるアニメーション（0.2秒程度）
- 基地HPが0になった瞬間：基地全体が一瞬光る（フラッシュ）演出
- 過度なエフェクトはStep1では不要。「動きが分かればOK」レベルで十分

## 機能仕様

### 操作
- 親：矢印キー（↑↓←→）で上下左右移動（8方向可）
- 子ども：WASDキーで上下左右移動（8方向可）

### 親（掃除機）の挙動
- おもちゃに接触すると自動で吸引する（ボタン操作不要。接触判定のみでよい）
- 容量上限（10）を超える場合は吸えない
- 吸ったおもちゃはインベントリとして保持し、移動速度は下記の計算式に従って低下する
- ゴミ箱に接触すると、保持しているおもちゃがすべて得点化され、インベントリが空になり容量・重さがリセットされる（移動速度も基本速度に戻る）

### 子ども（おもちゃ側）の挙動
- 床にあるおもちゃは、時間経過で自動的に得点としてカウントされ続ける（「床にある間だけ得点になる」という原則を必ず守ること）
- Step1では移動のみ（散らかす操作・妨害操作は実装しない。すでに散らかっている状態からスタートする）

### 基地（おもちゃ箱）
- マップ上に固定配置。中に初期状態でおもちゃ数個（大小混在）を格納した状態でスタートする
- 基地内のおもちゃは、基地が「開放」状態になるまで親は吸えない
- 親が基地に隣接した状態を継続すると、基地HPが1秒あたり10減少する
- 基地HPが0になったら「開放」状態になり、以後は通常のおもちゃと同様に親が吸える
- 修理キットを持った子どもが基地に隣接すると、基地HPが1秒あたり20回復する（上限100）
- 基地HPは画面上、基地の真上にゲージとして表示する

### 修理キット
- マップ上に1個、固定位置に配置（Step1範囲。将来的に基地内・フィールドの2箇所案あり、Step2以降で対応）
- 子どもが接触すると「所持」状態になる（見た目にキャラの色を変える等で所持が分かるようにする）
- 所持中に基地に隣接すると上記の回復効果が発生する

### 数値仕様（Step1固定値）

- 制限時間：120秒
- 掃除機の容量上限：10
- おもちゃの種類（Step1は2種類のみ）
  - 小：容量消費1、得点1、重さ寄与1
  - 大：容量消費5、得点10、重さ寄与10
- 移動速度の計算式：`速度 = max(0.2, 1 / (1 + 重さ合計 ÷ 10)) × 基本速度`
  - 基本速度は仮に「1秒間に画面幅の約1/4を移動できる速度」を基準値とする
- 基地HP上限：100（削れ方・回復速度は上記「基地」項目の通り）

### 得点・勝敗
- 時間終了時点で「親の得点（ゴミ箱に運んだ合計）」と「子どもの得点（床にあるおもちゃの合計、その時点のスナップショット）」を比較し、多い方が勝ち
- 結果画面に「親の勝ち／子どもの勝ち／引き分け」と、それぞれの最終得点を表示する

### エッジケース
- 親の容量がちょうど埋まっている状態で、容量ぴったりのおもちゃに接触した場合：吸える（容量オーバーにならない場合はOK）
- 修理キットを持った子どもが基地HP満タンの状態で基地に隣接し続けても、100を超えて回復しない（上限でクランプする）
- 制限時間が0になった瞬間に親または子どもがおもちゃに接触していても、その時点の得点計算は行わない（時間切れの瞬間で計算を打ち切る）
- 親のインベントリが空の状態でゴミ箱に接触しても、何も起こらない（エラーにならないこと）
